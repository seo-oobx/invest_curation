# 02. 제품 요구사항 정의서 (PRD)

## 1. 제품 개요
* **제품명:** Alpha Calendar (가칭)
* **플랫폼:** Web App (Mobile First, 반응형 디자인 필수)
* **타겟 유저:** 주식 투자를 하고 싶으나 분석할 시간이 없고, 언제 사고팔지 기준이 없는 2040 직장인.

## 2. 주요 기능 (Key Features)

### A. 메인 대시보드 (Curator)
* **기능:** 현재 투자 유효한(Active) 이벤트 리스트 카드 뷰 제공.
* **UI 구성:**
    * D-Day 카운트다운 (예: D-120)
    * 이벤트 제목 & 썸네일
    * Hype Gauge (화제성 점수를 시각적 게이지로 표현)
    * 태그: [Type A/B], [확정/예상], [카테고리]

### B. 이벤트 상세 페이지 (Analyst)
* **기능:** 특정 이벤트 클릭 시 상세 분석 데이터 제공.
* **데이터:**
    * **Hype Chart:** 일자별 관심도 변화 그래프 (검색량/댓글수 추이).
    * **Proxy Timeline:** 선행 시그널 발생 타임라인.
    * **Related Stocks:** 관련 종목 리스트 및 현재 주가/PBR 정보.
    * **Opinion:** "왜 지금인가?"에 대한 3줄 요약 (AI 자동 생성 or 관리자 입력).

### C. 알림 센터 (Alert)
* **기능:**
    * Hype Score 급상승 시 알림.
    * 선행 시그널(Proxy) 포착 시 알림.
    * D-Day 임박(D-30, D-7) 알림.

### D. 관리자(Admin) 페이지
* **기능:** 크롤링된 이벤트 데이터 검수 및 승인(Publish), Hype Score 수동 보정.

## 3. 유저 플로우 (User Flow)
1.  **Landing:** 서비스 가치 제안 -> 회원가입/로그인 (Supabase Auth).
2.  **Onboarding:** 관심 분야(바이오, 방산, 엔터 등) 선택.
3.  **Dashboard:** 큐레이션된 이벤트 리스트 확인.
4.  **Detail:** 관심 있는 이벤트 클릭 -> 상세 리포트 열람.
5.  **Action:** 관련 종목 확인 후 증권 앱으로 이탈(매매 하러 감) or 관심 등록.

## 4. 비즈니스 모델
* **Tier 1 (Free):** 기본 이벤트 리스트 열람.
* **Tier 2 (Pro):** Hype Chart 상세 보기, 선행 시그널 알림, 관련 종목 Valuation 리포트.
